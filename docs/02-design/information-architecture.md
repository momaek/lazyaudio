> 版本 v0.1 — 2026-05-16
> 配套 [`../01-research/prd.md`](../01-research/prd.md)
> 本文档定义页面 / 浮窗 / 菜单的清单和层级关系，不做视觉决策。

## 1. 顶层结构

LazyAudio 是**单窗口 App**，配**菜单栏常驻图标**。所有功能都从这两处入口进入。

```
LazyAudio
├── 菜单栏图标（macOS top-bar / Windows tray）       — 常驻、轻量
├── 主窗口（独立、可关闭，关闭=最小化到菜单栏）       — 重操作
├── 录音前浮窗（modal-like，全局快捷键 / 菜单栏触发） — 1 步交互
└── Onboarding 窗口（首启时唯一窗口；走完才显示主窗口）
```

## 2. 菜单栏图标

### 2.1 视觉状态（3 种）

| 状态 | 显示 | 触发 |
|---|---|---|
| 空闲 | 仅 App 图标 | 未在录音 |
| 录音中 | `● 03:24`（红点呼吸 + 时长） | 录音进行中 |
| 暂停 | `‖ 03:24`（红点不脉冲，灰化） | 用户暂停录音 |

### 2.2 点击展开的 dropdown

**空闲时**：
```
开始录音…           ⌘⇧R
─────────────────
显示主窗口          ⌘1
最近录音 ▸
  ├ 面试官 2026-05-16 14:30
  ├ 会议 2026-05-15 10:00
  ├ 笔记 2026-05-15 09:12
  ├ 会议 2026-05-14 15:00
  └ 在主窗口中查看全部…
─────────────────
设置…              ⌘,
退出 LazyAudio     ⌘Q
```

**录音中**：
```
● 03:24  录制中（{会话类型}）
─────────────────
暂停录音           ⌘⇧P
停止并保存         ⌘⇧R
─────────────────
显示主窗口          ⌘1
─────────────────
设置…              ⌘,
退出 LazyAudio…    ⌘Q   ← 提示"录音将停止"
```

设计要点：
- 录音中点 `开始录音` 等同于 `停止并保存`（同一个全局快捷键的双向语义）
- 最近录音最多 5 条；点击 = 打开主窗口并定位到该条
- "退出 LazyAudio" 在录音中变灰 + 提示文字，需二次确认

## 3. 主窗口

### 3.1 整体布局

**Chrome-less 设计** — 无独立标题栏 chrome 行；traffic light 嵌入 sidebar 顶部。详见 [`design-system.md §10.1`](./design-system.md)。

```
┌──────────────────┬───────────────────────────────────────────────┐
│ ● ● ●        ⚙   │  标题（可双击编辑）            [↻][⬇][⋯]    │ ← 详情头紧贴窗口顶部
│ ─────────────── │  类型徽章 · 时长 · 时间 · 转录状态            │
│ [🔍 搜索全文…  ] │                                                │
│ [全部][会议][笔记]│  ┌──────────────────────────────────────────┐ │
│                  │  │ 〰〰〰〰〰〰 波形 + 时间轴 〰〰〰〰〰〰    │ │ ← 播放器
│ — 今天 ——————   │  │   ▶ 00:14 / 01:23:45    [-15s][+15s][1×] │ │
│ ● 面试官 14:30   │  └──────────────────────────────────────────┘ │
│   01:12:34       │                                                │
│   "好的我们先…"   │  ┌─ 转录 ─────────────┬─ AI 摘要 ─────────┐  │
│                  │  │ [00:00] 🎤 你好... │  会议纪要          │  │
│ ○ 会议 10:00     │  │ [00:08] 🔊 你好... │  ─────────         │  │
│   00:45:11       │  │ [00:15] 🎤 我们... │  议题：…           │  │
│   "我们今天要…"   │  │ ...                 │  待办：…           │  │
│                  │  │                     │  决议：…           │  │
│ — 昨天 —————    │  │                     │                    │  │
│ ○ 笔记 16:45     │  │                     │  [重新生成] [复制] │  │
│   00:01:23       │  └─────────────────────┴────────────────────┘  │
│ ...              │                                                │
└──────────────────┴───────────────────────────────────────────────┘
```

- **窗口高 800px**，无 chrome 行 → 内容区从 y=0 开始
- **Sidebar 顶部 36px** 为 traffic light 行（同 sidebar 灰底）：左 traffic light（macOS 三圆点 / Windows window controls），右 ⚙ 设置入口
- **App 名 "LazyAudio" 不显示**（dock / menu bar 已有）
- **窗口拖拽**：traffic light 行 `-webkit-app-region: drag`

### 3.2 区域清单

| 区域 | ID | 内容 | 状态 |
|---|---|---|---|
| Traffic light 行 | traffic-row | macOS 三圆点 / Win window controls + ⚙ 设置入口 | 嵌在 sidebar 顶部 36px |
| 库侧边栏 | sidebar | 搜索 + 类型 chip + 列表（traffic light 行在最顶） | 可隐藏 |
| 列表分组头 | list-group-header | "今天 / 昨天 / 本周 / 本月 / 更早" | 自动按日期 |
| 列表项 | list-item | 类型徽章 + 标题 + 时长 + 转录预览首句 | 单选高亮 |
| 详情头 | detail-header | 标题（可编辑）+ 元信息 + 操作 | 右上一组 icon |
| 播放器 | player | 波形 + 时间 + 控制按钮 | 暂停 / 倍速 / 跳转 |
| 转录面板 | transcript | 段落列表，每段：时间戳 + speaker + 文字 | 可点击跳转 |
| 摘要面板 | summary | 模板选择 + 摘要内容 + 操作 | 可折叠 |
| 空状态 | empty | 无录音时的引导 | onboarding 后第一次进入 |

### 3.3 列表项操作

右键 / 长按 / 三点菜单：
- 重命名（行内编辑）
- 复制转录全文
- 导出（md / txt / srt 子菜单）
- 在 Finder / Explorer 中显示
- 重新转录
- 重新生成摘要
- 删除…（带确认 dialog）

### 3.4 详情头操作

右上一组：
- `[⬇ 导出 ▾]` — md / txt / srt 下拉
- `[↻ 重新生成]` — 重转 / 重摘要
- `[⋯]` — 在 Finder 显示 / 删除

### 3.5 列表项的转录状态

| 状态 | 视觉 | 行为 |
|---|---|---|
| 录音中（Pass A 跑中） | 红点脉冲 `●` 替代徽章；列表项预览首句实时刷新为最新 hypothesis | 列表项可点击进入详情查看实时转录 |
| 录音中（Pass A 关闭 / 资源受限） | 红点脉冲 `●` 替代徽章；列表项预览空 | 列表项可点击但详情转录区显示 "录音中（Pass A 已关闭）" |
| 转录排队（Pass B 等待中） | 灰色 `…` + 类型徽章 | 详情显示 Pass A 段落（hypothesis/confirmed 视觉），底部 progress 条 |
| 转录中（Pass B 跑中） | 旋转 `⟳` + 进度数字（如 `34%`） | 详情显示 Pass A 段落 + Pass B progress |
| 转录完成（Pass B 已覆盖） | 类型徽章 + 详情头 `✓ 离线精修` 标记 | 完整可用 |
| 转录失败（Pass B 挂） | 红色 `!` | 详情仍显示 Pass A 段落（保底可用），底部错误 + 重试按钮 |

## 4. 录音前确认浮窗

### 4.1 触发

- 全局快捷键（默认 `⌘⇧R` / `Ctrl+Shift+R`）
- 菜单栏 `开始录音…`
- 主窗口某按钮（位置 02-design mockup 决定）

### 4.2 内容

```
┌─ 准备录音 ─────────────────────────┐
│                                     │
│  会话类型  [面试官 ▾]               │ ← 默认 = 上次 / 通用
│                                     │
│  音源     [✓] 麦克风                │
│           [✓] 系统音                │
│                                     │
│  [取消]              [开始录音 →]    │
│                                     │
│  ⌘⇧R 直接开始        ⌘. 取消        │
└─────────────────────────────────────┘
```

### 4.3 行为

- 默认聚焦"开始录音"按钮，Enter 直接开始
- Esc / ⌘. 取消
- 浮窗位置：macOS 屏幕中心 / 触发位置就近（02-design 再定）
- 设置中可勾选"快捷键直接开始录音，跳过浮窗"

## 5. 设置窗口

独立窗口（不是主窗口的 tab），左侧 nav + 右侧 content。

```
┌─ 设置 ──────────────────────────────────────────┐
│ 通用       │ 通用                                │
│ 录音       │ ─────                               │
│ 转录引擎   │ [✓] 开机启动                        │
│ LLM 模板   │ [✓] 关闭主窗口时最小化到菜单栏      │
│ 快捷键     │ 默认会话类型  [通用 ▾]              │
│ 隐私       │ App 语言      [中文 ▾]              │
│ 关于       │ 外观          [跟随系统 ▾]          │
└────────────┴─────────────────────────────────────┘
```

### 5.1 各 tab 内容

| Tab | 设置项 |
|---|---|
| 通用 | 开机自启 / 关闭主窗口行为 / 默认会话类型 / 语言 / 外观（亮 / 暗 / 跟随） |
| 录音 | 保存目录 / 是否生成分轨 / 是否生成混音 / 文件命名格式 / 录音结束自动转录开关 |
| 转录引擎 | 本地 / 云端切换；本地：模型列表（已下载 + 可下载 + 占用空间）；云端：baseUrl / apiKey / model 字段 |
| LLM 模板 | 5 个预置模板的列表 + 各自的 prompt 编辑框；按会话类型自动套用的映射表 |
| 快捷键 | 开始/停止录音、暂停、显示主窗口；点击 = 录新快捷键 |
| 隐私 | 录音合规提示开关 / 错误上报（v0.1 关，灰显） |
| 关于 | 版本号 / 开源协议 / 模型协议 / 致谢 |

## 6. Onboarding 窗口

仅首启时显示，**模态全屏感** —— 不允许跳过权限和必要配置。

### 6.1 步骤序列

```
[1 欢迎] → [2 隐私模式] → [3 权限] → [4a 模型下载 | 4b API 配置] → [5 快捷键] → [6 合规提示] → [7 完成]
```

### 6.2 每步内容

| # | 标题 | 内容 | 可跳过 | 退出后果 |
|---|---|---|---|---|
| 1 | 欢迎使用 LazyAudio | 一句话定位 + 大图 | 否 | — |
| 2 | 选择隐私模式 | 本地（推荐）/ 云端 单选 | 否 | 决定 4a / 4b 走向 |
| 3 | 授权权限 | macOS：麦克风（仅）/ Windows：麦克风 | 否（拒绝则进入降级页） | 必须授权才能录音 |
| 4a | 下载本地模型 | 默认 SenseVoice，进度 / 取消 / 重试 / 切换镜像源 | "稍后下载"可跳过，但不能录音 | 完成后才解锁主窗口 |
| 4b | 配置 OpenAI 兼容 API | baseUrl / apiKey / model，测试连接 | "稍后填"可跳过 | 完成后才能用云端 |
| 5 | 确认快捷键 | 显示默认 `⌘⇧R`，可改 | 是（用默认） | — |
| 6 | 录音合规提示 | 一段文字 + "我已知晓"按钮 + "下次不提示"勾选 | 否（必须点已知晓） | — |
| 7 | 完成 | "开始你的第一次录音" + 提示快捷键 | — | 关闭 → 主窗口 |

### 6.3 异常分支

- 第 3 步用户拒绝麦克风：进入**降级页**，告知"无麦克风权限无法录音"，提供"打开系统设置"按钮 + "稍后再说"
- 第 4a 步下载失败：保留进度，提供"切换镜像 / 重试 / 跳过（暂时不能录音）/ 切到云端"
- 第 4b 步 API 测试失败：标红 + 错误信息，可"跳过"以后再填

## 7. 全局元素

跨页面复用的视觉元件，design-system.md 定具体样式。

### 7.1 类型徽章（session type badge）

7 种，每种一个 icon + 颜色 + 中文 label：

| 类型 | label | icon 候选 | 颜色家族 |
|---|---|---|---|
| general | 通用 | • | 灰 |
| meeting | 会议 | 👥 / 🗂 | 蓝 |
| note | 笔记 | ✎ | 黄 |
| interview-as-interviewer | 面试官 | 👤→ | 紫 |
| interview-as-candidate | 面试者 | →👤 | 粉 |
| lecture | 课程 | 🎓 | 绿 |
| podcast | 播客 | 🎙 | 橙 |

### 7.2 转录状态徽章

`pending` / `running` / `done` / `failed` — design-system.md 定颜色。

### 7.3 呼吸红点

录音中的菜单栏指示 + 列表项指示：

- 颜色：录制红
- 周期：1.2s
- 透明度：60% ↔ 100%
- ease-in-out

### 7.4 speaker 标识

转录文本每段开头一个小 icon：

- 🎤 = mic（用户）
- 🔊 = system（对方 / 系统音）

具体视觉（颜色 / 缩进 / 头像位）design-system.md 定。

### 7.5 时间戳锚点

`[00:14]` 格式，点击 → 音频跳转。hover 高亮、当前播放段加粗。

## 8. 通知（系统级）

| 触发 | 内容 | 行为 |
|---|---|---|
| 转录完成 | "{标题} 转录完成" | 点击 → 主窗口对应录音 |
| 转录失败 | "{标题} 转录失败，点击查看" | 点击 → 主窗口 + 错误详情 |
| 模型下载完成 | "本地转录模型已就绪" | 仅首启场景 |
| 长录音提醒（>2h，可选 v0.2） | "你已录制 2 小时，是否继续？" | 不打断录音 |

通知可在设置中分类关闭（v0.2，v0.1 不做）。
