## 1. 产品一句话

**LazyAudio 是一个本地优先的桌面应用（macOS 14.2+ / Windows 10+），录下电脑系统音和麦克风音频，自动转录成可搜索的文字，并能用 LLM 一键生成会议纪要 / 面试评估 / 课程笔记。**

---

## 2. 用户与场景

第一批用户 = 技术 / 产品 / 设计岗的个人贡献者或 manager，每周大量会议 + 1-on-1 + 面试，**注重隐私，不愿把会议内容传第三方云**。

核心使用场景：

1. 线上会议（Zoom / Teams / 腾讯会议）— 录 + 转录 + 生成纪要
2. 语音笔记 / 灵感速记 — 快捷键秒起
3. 作为面试官的面试 — 复盘候选人回答、生成结构化评估
4. 作为面试者的面试 — 自我复盘、记录卡壳点
5. 课程 / 讲座录制 — 章节笔记
6. 播客录制（弱）— 分轨保留

---

## 3. 设计调性（关键约束）

**关键词**：沉稳、专注、可信任、工具感、信息密度高。

**参考**：macOS 系统设置、Linear、Logic Pro 录音面板、Obsidian / Tana 的工具感。

**禁止**：

- ❌ 装饰性渐变背景 / 玻璃拟态卡片 / 大量阴影 / 装饰性图形
  - 唯一例外：macOS 系统原生 vibrancy 可用于浮窗 / dropdown 背景（见 §5.6）；不要在主窗口内部、卡片、按钮上使用磨砂
- ❌ 卡通插画 / 营销感 banner
- ❌ 过度动画（除录制脉冲、加载旋转）
- ❌ "立即升级" 这类消费 SaaS 文案
- ❌ **顶部 chrome 标题栏 / App 名标识 / 独立 window controls 行** —— 现代 macOS app（Linear / Arc / Notion / Cursor）都已抛弃 old-school 的"顶部 chrome + App 名"模式（详见 §4.1）

**视觉特征**：

- ✅ 大量灰阶 + 极少强调色
- ✅ 紧凑信息密度（参考 Linear / Obsidian）
- ✅ 录制状态（呼吸红点）是全 App 唯一的"花"
- ✅ 深色模式 / 浅色模式跟随系统

## 4. 核心交互形态

**单主窗口 + 菜单栏常驻图标**。

- 菜单栏图标显示录音状态（`● 03:24` 红点呼吸 + 时长）
- 全局快捷键 `⌘⇧R`（macOS）/ `Ctrl+Shift+R`（Windows）一键开始 / 停止录音
- 主窗口左侧录音库（按日期分组）+ 右侧详情（播放器 + 转录 + 摘要）

### 4.1 Chrome-less 窗口（重要约束）

主窗口 / 设置窗口 **不画独立的标题栏 chrome 行**：

```
✗ 错误做法（old-school）：
┌──────────────────────────────────────┐
│ ● ● ●        LazyAudio          ⚙ │ ← 独立 44px chrome 行
├──────────────┬───────────────────────┤
│  sidebar     │  content              │
└──────────────┴───────────────────────┘

✓ 正确做法（chrome-less）：
┌──────────────┬───────────────────────┐
│ ● ● ●    ⚙ │  content 立即开始     │ ← traffic light 嵌 sidebar 顶部 36px
├──────────────┤                       │
│  sidebar     │                       │
└──────────────┴───────────────────────┘
```

- 不显示 App 名（dock / menu bar 已有）
- Traffic light（macOS 三圆点）直接嵌入 sidebar 顶部，**与 sidebar 同灰底**
- 设置 ⚙ icon 放在同一行右侧
- 整个 traffic light 行 36px 高，padding `[0, 12]`，`-webkit-app-region: drag` 作为拖拽区
- 内容区从窗口 y=0 开始，无 chrome 偏移
- 录音中红色 banner、错误 banner 等**只跨内容区**（从 x=sidebar-width 开始），不跨 sidebar，保持 traffic light 区域始终干净

## 5. 视觉规范摘要

### 5.1 色板

```
中性灰阶（gray-0 / gray-50 → gray-1000，12 级；gray-0=#fff，gray-1000=#0A0C0F）
accent  #1F6FEB（浅）/ #4493F8（深）   - 主色，按钮 + 链接 + 选中
record  #E5484D / #FF6369              - 仅录制状态使用
success #3E9B4F / #46A758
warning #D97706 / #F2A742
danger  #DC2626 / #FF6369
```

### 5.1.1 表面层级（surface layers）

UI 由若干灰阶层叠出来，请严格按以下分配，不要乱用：

| Token              | 浅色          | 深色         | 用途                         |
| ------------------ | ------------- | ------------ | ---------------------------- |
| `surface/canvas`   | gray-50       | gray-950     | 窗口最底层背景（左侧列表区） |
| `surface/panel`    | gray-0 (#fff) | gray-900     | 右侧详情区、设置内容区       |
| `surface/raised`   | gray-0        | gray-850     | 浮窗、dropdown、悬浮卡片     |
| `surface/sunken`   | gray-100      | gray-1000    | 转录段落 hover、输入框 bg    |
| `surface/hover`    | gray-100      | gray-850     | 列表项 hover                 |
| `surface/selected` | accent @ 8%   | accent @ 12% | 列表项选中、当前播放段       |
| `border/subtle`    | gray-200      | gray-800     | 分割线、面板边界             |
| `border/strong`    | gray-300      | gray-700     | 输入框、按钮边框             |

原则：相邻两层亮度差 ≥ 1 级，绝不出现"两个 panel 贴在一起分不清"。

### 5.1.2 Speaker 色（转录段落 speaker 标识专用）

按说话人出现顺序循环分配，**不要和 §5.2 会话类型色混用**，否则用户会分不清颜色代表什么。

| Token        | 浅色     | 深色     | 用于                                  |
| ------------ | -------- | -------- | ------------------------------------- |
| `speaker-1`  | #1F6FEB  | #4493F8  | 第 1 位说话人（通常是用户，麦克风源） |
| `speaker-2`  | #D97706  | #F2A742  | 第 2 位说话人（通常是系统音源）       |
| `speaker-3`  | #3E9B4F  | #46A758  | 第 3 位说话人                         |
| `speaker-4`  | #B85C97  | #DD7CC4  | 第 4 位说话人                         |
| `speaker-5+` | gray-500 | gray-500 | 5 人以上一律灰，不再分配              |

转录段中常见缩写 `●M` = speaker-1（Microphone）、`●S` = speaker-2（System）。

### 5.2 7 种会话类型色（仅用于徽章 + 列表项 icon）

| sessionType              | 中文   | hue     |
| ------------------------ | ------ | ------- |
| general                  | 通用   | gray    |
| meeting                  | 会议   | blue    |
| note                     | 笔记   | yellow  |
| interview-as-interviewer | 面试官 | purple  |
| interview-as-candidate   | 面试者 | pink    |
| lecture                  | 课程   | emerald |
| podcast                  | 播客   | orange  |

### 5.3 字体

- UI：SF Pro / Segoe UI / PingFang SC 系统字
- 转录文本 + 时间戳：SF Mono / monospace 13px

### 5.4 间距 / 圆角

- 间距：4 / 8 / 12 / 16 / 24 / 32 / 48
- 圆角：徽章 4px / 输入框按钮 6px / 列表项 8px / 浮窗 12px

### 5.4.1 控件高度（统一 token，跨页面强制一致）

| 控件                              | 高度 | 内边距 | 备注                                                |
| --------------------------------- | ---- | ------ | --------------------------------------------------- |
| 主按钮 / primary                  | 32px | 12 横  | 文字 13px semi-bold，accent bg，hover 加深 4%       |
| 次按钮 / secondary                | 32px | 12 横  | `surface/raised` bg + `border/strong`，文字 13px    |
| 小按钮 / compact                  | 24px | 8 横   | 详情头 / 工具栏内，文字 12px                        |
| 输入框                            | 32px | 8 横   | 配 `border/strong`，focus 边框变 accent             |
| 大输入框（onboarding / 设置表单） | 36px | 12 横  | API 配置等                                          |
| segmented control                 | 28px | —      | 整体 6px 圆角；单段平角，选中段 `surface/raised` bg |
| 链接按钮                          | 行内 | —      | 无 bg，accent 文字，hover 下划线                    |

### 5.5 主窗口尺寸

- 默认 1280×800
- 最小 960×600
- 列表项默认 56px 紧凑模式（一行标题 + 一行预览）

### 5.6 跨平台差异（macOS 14.2+ vs Windows 10+）

| 项              | macOS                                      | Windows                             |
| --------------- | ------------------------------------------ | ----------------------------------- |
| 窗口控件位置    | 左上 traffic light（红黄绿圆点）           | 右上 `— □ ×`                        |
| 标题栏          | **见 §5.7**（无独立标题栏，集成式 chrome） | **见 §5.7**（自绘无 chrome 标题栏） |
| 字体            | SF Pro / SF Mono                           | Segoe UI / Cascadia Mono            |
| 快捷键键帽      | `⌘ ⇧ ⌥ ⌃` 符号                             | `Ctrl / Shift / Alt / Win` 文字     |
| 菜单栏常驻      | 顶部系统菜单栏右侧状态栏 icon              | 系统托盘 icon（右下角）             |
| 浮窗 / 弹窗阴影 | macOS 标准 vibrancy（轻磨砂）              | Windows Acrylic（弱阴影，无磨砂）   |
| 滚动条          | 自动隐藏，hover 出现                       | 始终可见，10px 宽                   |
| 按钮 hover      | 整体微亮 4%                                | 加 1px accent 描边                  |

> 默认按 macOS 形态画。Windows 版**只换窗口外壳 + 上述控件细节**，主窗口内部布局、间距、组件、字号一律一致——不要做两套不同的信息架构。

### 5.7 窗口外壳（chrome）— 全局规则

**所有 App 窗口**（主窗口 / 设置窗口 / onboarding 窗口 / 录音前浮窗）一律采用**现代化集成式 chrome**，参考 Linear / Arc / Notion / Things 3 / macOS 14 Settings。

**硬性规则**：

- ✅ macOS：整窗用 `fullSizeContentView`，标题栏与内容融合
- ✅ Windows：自绘无 chrome 标题栏，整窗 client area 占满
- ✅ 窗口控件浮在内容层之上：
  - **macOS** traffic light：浮在窗口左上角，距上 **18px**、距左 **16px**，位于侧边栏区域内部，下方不留多余间隙
  - **Windows** `— □ ×`：浮在窗口右上角，距上 **8px**、距右 **8px**
- ✅ **不显示窗口标题文字**——App 名归 macOS 系统菜单栏 / Windows 任务栏管，窗口内不重复 "LazyAudio"
- ✅ 顶部 12–16px 高视为**隐式拖拽区**（无视觉边框，无背景色），整片可拖窗
- ✅ 内容直接从顶部开始：侧边栏第一个元素（搜索框 / 进度指示器）顶到 traffic light 同高，左侧用 16px 内边距把它推开

**严禁**：

- ❌ 画独立的标题栏行（即图里 `┌─────┐` 下面再加一条 `│ LazyAudio ⚙ │ ─ □ × │` 的横带）
- ❌ ASCII 草图里的 `┌─ XX ──┐` 带居中标题写法
- ❌ traffic light 和首行内容之间留 ≥ 20px 的"标题栏高度"空白
- ❌ 浮窗 / modal 用 `┌─ 标题 ─┐` 这种把标题烤进上边框的 2015 式样

**浮窗 / modal 的题头**（如录音前浮窗、二次确认 modal）：标题作为**内容的第一行**而非边框的一部分——17px semi-bold + 右上角 `×` 关闭按钮（24px compact），与下方表单留 16px 间距。

**录音中顶部状态条**（§6.3.5）：record 色 @ 8% 胶囊条**浮在 traffic light 右侧的拖拽区内**，不顶出额外的一行；高 24px，左缘距 traffic light 12px，宽度自适应文字。

## 6. 必须画的页面（按优先级）

### P0 — 核心体验闭环

#### 6.1 主窗口（默认有数据状态）

布局：左侧库列表 + 右侧详情（播放器 + 转录 + 摘要）。**窗口外壳严格遵守 §5.7**——无独立标题栏，traffic light 浮在左上角。

```
┌──────────────────┬──────────────────────────────────────────┐
│ ⊙⊙⊙              │                                ⚙   🌗     │  ← 12–16px 拖拽区，无视觉边框
│ 🔍 搜索…  ⌘F      │  面试官 2026-05-16 14:30      [↻][⬇][⋯] │
│ 全部 通用 会议 …  │  🏷 面试官 · 01:12:34 · 已完成           │
│                  │                                          │
│ ─ 今天 ─────     │  〰〰〰〰〰〰 波形 〰〰〰〰〰〰              │
│ ● 面试官 14:30   │  ▶ 00:14 / 01:23:45    [-15s][1×][+15s] │
│   01:12:34       │                                          │
│   "好的我们先…"   │  ┌──────────────┬─────────────────────┐ │
│                  │  │  转录         │  AI 摘要 ───────   │ │
│ ○ 会议 10:00     │  │ [00:00] ●M 你好│  会议纪要         │ │
│   00:45:11       │  │ [00:08] ●S 你好│  议题：…          │ │
│                  │  │ [00:15] ●M 我们│  待办：…          │ │
│ ─ 昨天 ─────     │  │ ...           │  决议：…          │ │
│ ○ 笔记 16:45     │  │               │                   │ │
│   00:01:23       │  │               │  [换模板] [复制]  │ │
│                  │  └──────────────┴─────────────────────┘ │
└──────────────────┴──────────────────────────────────────────┘
```

> **注意**：上图最顶行 `⊙⊙⊙ ... ⚙ 🌗` 是浮层，**不是独立标题栏**——它和下方 "🔍 搜索…" / "面试官 2026-05-16 14:30" 共占同一个集成式顶部区域，无横向分隔线。Windows 版 `⊙⊙⊙` 换成右上角 `— □ ×`。

要点：

- 列表项：左侧类型 icon + 颜色点 → 标题 → 时长 → 转录预览首句
- 列表分组头："今天 / 昨天 / 本周 / 本月 / 更早"
- 列表项录音中：圆点用呼吸红点 + 标题加 "（录制中）"
- 详情头标题可双击编辑
- 转录每段：`[时间戳] ●speaker 文字`，speaker 用 §5.1.2 token 轮换分配（`●M` = `speaker-1` / `●S` = `speaker-2`），**不要复用会话类型色**
- 摘要面板可折叠，默认展开

尺寸细节（画稿时必须遵守）：

| 区域               | 尺寸      | 备注                                                                                     |
| ------------------ | --------- | ---------------------------------------------------------------------------------------- |
| 左侧列表栏宽       | 280px     | 可拖拽 240–360                                                                           |
| 详情头             | 高 56px   | 标题 + 类型徽章 + 操作按钮一行。和列表项同高纯属功能巧合，**不要刻意把左右栏对齐成网格** |
| 波形条             | 高 64px   | gray-300 描线，已播放段填 accent                                                         |
| 播放控制条         | 高 40px   | 播放/暂停按钮 32px，速度/跳转 28px                                                       |
| 转录 / 摘要分隔    | 50% / 50% | 中间 1px `border/subtle`，可拖拽 30–70%                                                  |
| 转录段落行高       | 24px      | 段间距 8px                                                                               |
| 摘要面板底部操作条 | 高 44px   | `[换模板]` `[复制]` 右对齐                                                               |

**搜索 + 类型筛选**（列表栏顶部，详情上方）：

- **搜索框**：列表栏顶部 32px 高 input，placeholder `搜索录音 / 转录…`，`⌘F` 聚焦
- 输入时左侧列表实时过滤；命中转录正文时，列表项预览替换为命中片段 + 高亮（黄底 `<mark>`）
- **类型筛选 chip 行**：搜索框下方 32px 高一行 chip，"全部 / 通用 / 会议 / 笔记 / 面试官 / 面试者 / 课程 / 播客"
  - 单选，默认"全部"
  - 选中态：chip bg = 类型色 @ 12%，文字 = 类型色，1px 类型色边框
  - 非选中态：bg `surface/sunken`，文字 gray-700/gray-300
  - chip 高度 24px，横向 padding 8px，圆角 12px（胶囊）
  - 超出列表栏宽度时横向滚动（隐藏滚动条），不要换行
- **空搜索结果**：列表区显示居中灰字"没有匹配的录音" + "清除搜索"链接

#### 6.2 主窗口 — 空状态

首次进入 + 主区域显示居中插画 / 引导（不用卡通插画，用简洁线条 + 文字）：

```
还没有录音
按 ⌘⇧R 开始第一次录音

[开始录音]   ← 主按钮
```

#### 6.3 主窗口 — 转录中 / 失败状态

列表项 + 详情两个状态都要表达：

- 列表项：徽章替换为 `⟳ 34%` pending 色
- 详情：转录文本区显示骨架屏 + 进度，摘要面板 disabled

失败：列表项徽章红 `⚠ 转录失败`，详情区显示 `"网络连接超时"` + `[重试]` 按钮。

#### 6.3.5 主窗口 — 录音中状态

用户保持主窗口打开期间录音时，主窗口与菜单栏（§6.5）状态同步：

- **左侧列表**：当前会话置顶（无视日期分组），圆点替换为 §7.2 呼吸红点，标题后跟灰字 "（录制中）"，时长实时跳秒（monospace）
- **详情头**：类型徽章右侧追加 `⏸ 暂停` `⏹ 停止` 两个 32px 圆角按钮（替换原本 §6.1 的 `[↻][⬇][⋯]` 操作组）
- **波形区**：实时滚动，新采样从右向左推入，已采集段填 record 色 @ 60%，幅度由实时音量决定；播放控制条整行 disabled
- **转录区（Multi Pass 实时显示，v0.1 P0）**：
  - 录音中**实时追加段落**（Pass A 输出）——hypothesis 段用**灰色斜体**（表达"尚未最终确认"），几秒后被 confirmed 版本**原地替换**为正常正文样式
  - segment id 稳定（spike-013 验证），原地替换时阅读光标 / 滚动位置不跳
  - 录音 stop → Pass B 开始：详情区出现细灰色 progress 条 + 灰字 "正在生成离线高精度版"（不挡内容、不切视图）
  - Pass B 完成 → segments **整体覆盖** Pass A，UI 原地刷新，**不弹通知**，progress 条消失
  - 设置中可关 Pass A（菜单栏 only / 性能受限机器场景）
- **摘要面板**：disabled，灰文字 "录音结束后生成"，`[换模板]` `[复制]` 按钮置灰
- **顶部状态条**：标题栏正下方固定一行 28px 高状态条，record 色 @ 8% 背景 + 上下 1px record 色 @ 20% 描边；左侧 `● 正在录音 — 03:24（面试官）`（呼吸红点 + monospace 时长），右侧 `[暂停] [停止]` 12px 文字按钮；点击行任意位置聚焦到详情头控件
- **中途离线提醒 banner**（PRD F4.8，录制满 10/20/30… 分钟时）：状态条下方插一行 28px 高 info banner：`已录 N 分钟。对前 N 分钟跑离线精修？[跑离线] [稍后]`。点 [跑离线] → 后台增量 Pass B（仅 stop 时间段），不中断录音；点 [稍后] → banner 隐藏 10 分钟后再提

## 7. 关键组件细节

### 7.1 类型徽章

```
紧凑（列表项）：  ● 颜色点 4px
标准（详情头）：  [icon + 类型中文] 圆角矩形，类型色 @ 10% bg，类型色文字
```

### 7.2 呼吸红点

- 8px 圆，录制红
- 1.2s 周期，opacity 60% ↔ 100%，scale 0.92 ↔ 1.0，ease-in-out infinite
- 暂停态：动画停 + 颜色变 gray-400

### 7.3 转录段落

```
[00:14]  ●M  你好我们今天讨论一下项目进度
         ─── 11px monospace gray-500
              ── 圆点 4px + 字母 M/S 10px monospace
                   ── 文字 13px monospace text-base
```

- 当前播放段：背景 accent @ 8%，时间戳变 accent 色加粗
- 段落间距：8px

## 8. 关键交互动效

| 交互         | 效果                       | 时长            |
| ------------ | -------------------------- | --------------- |
| 浮窗入场     | scale 0.96→1 + opacity 0→1 | 100 ms ease-out |
| 列表项 hover | bg → `surface/hover`       | 80 ms           |
| 录制红点     | 呼吸（见 7.2）             | 1200 ms         |
| 转录 loading | 圆环旋转                   | 800 ms linear   |

不做：页面切换 slide、stagger 列表进入、鼠标跟随。
