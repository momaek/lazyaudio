## 1. 对话框（Modal Dialogs）

### 1.1 通用规范

| 项   | 值                                                                |
| ---- | ----------------------------------------------------------------- |
| 尺寸 | 440×220（标准） / 440×260（多按钮） / 520×280（含说明）           |
| 圆角 | 12                                                                |
| 背景 | `surface/raised`（white / gray-850）                              |
| 阴影 | `0 8px 32px rgba(0,0,0,0.20)`（强阴影，从背景拉开）               |
| 遮罩 | 父窗口 black @ 30% + blur 4px（macOS）/ 仅 black @ 40%（Windows） |
| 居中 | 屏幕居中                                                          |
| 入场 | 100ms ease-out，scale 0.96 → 1，opacity 0 → 1                     |
| 退场 | 80ms ease-in                                                      |
| 键盘 | Enter = 默认按钮；Esc = 取消按钮                                  |

### 1.2 布局

```
┌─ Title bar (高 44, padding [0, 20]) ─────────┐
│  标题文字 13 medium                    [×]  │
├──────────────────────────────────────────────┤
│ Body (padding 20, gap 12)                    │
│  ⚠ icon  正文 13 gray-700 line-height 1.55  │
│                                              │
│  (可选辅助文字 / 列表)                        │
├──────────────────────────────────────────────┤
│ Footer (高 56, padding [0, 20])              │
│                          [取消] [主操作]      │
└──────────────────────────────────────────────┘
```

- 左 icon 24px，根据语义：warning / danger / info
- 主操作右对齐，是 primary 或 danger primary
- 取消是 secondary

### 对话框 D1 — 删除录音确认

**触发**：列表项删除 / `Del` 键 / 右键 → 删除

**视觉**：

| 区     | 内容                                          |
| ------ | --------------------------------------------- |
| 标题   | `删除录音？`                                  |
| icon   | lucide `trash-2` 24 danger                    |
| 正文   | `「{标题}」将被永久删除，包括：`              |
| 列表   | - 音频文件 ({size})\n - 转录文本\n - LLM 摘要 |
| 强调   | `这个操作不可恢复。` 13 danger medium         |
| Footer | `[取消]` secondary ｜ `[删除]` danger primary |

**默认 focus**：`取消` 按钮（破坏性操作不默认 destructive）。

**键盘**：Enter = 取消，Esc = 取消；删除必须显式点击。

**复数情况**：用户多选时（v0.2）：`确认删除 N 条录音？将永久删除 {总大小}`，列表改为前 3 条标题 + `... 共 N 条`。

### 对话框 D2 — 录音中退出 App 确认

**触发**：`⌘Q` 录音中 / 菜单栏 `退出 LazyAudio…` 录音中。

**视觉**：

| 区     | 内容                                                                            |
| ------ | ------------------------------------------------------------------------------- |
| 标题   | `录音正在进行`                                                                  |
| icon   | lucide `mic` 24 record                                                          |
| 正文   | `当前有一段录音正在进行（已录 {duration}）。退出 LazyAudio 将：`                |
| 列表   | - 立即停止当前录音\n - 保存已录制的部分到本地\n - 启动转录（在 App 重启后继续） |
| Footer | `[继续录音]` primary ｜ `[停止并退出]` danger outline                           |

**默认 focus**：`继续录音`（更安全）。

**键盘**：Enter = 继续录音，Esc = 继续录音。

### 对话框 D6 — 磁盘空间不足

**触发**：录音前检查 / 模型下载前检查，可用空间 < 500 MB。

**视觉**：

| 区     | 内容                                                         |
| ------ | ------------------------------------------------------------ |

## 2. 系统通知（OS-level Notifications）

### 2.1 通用规范

- **macOS**：通过 macOS Notification Center 投递，遵循系统视觉
- **Windows**：通过 Toast Notifications，遵循系统视觉
- **不能自定义视觉，只能定义**：
  - icon（App icon）
  - 标题（约 60 字符上限）
  - 副标题（约 100 字符上限）
  - 主体文字（约 200 字符）
  - 1-3 个 action button
- **行为**：
  - 点击通知主体 → 打开主窗口 + 定位到关联录音
  - 点击 action 按钮 → 执行对应动作

### 2.2 通知清单

#### N1 — 转录完成

| 字段    | 内容                  |
| ------- | --------------------- |
| 标题    | `{标题} 转录完成`     |
| 副标题  | `{时长} · {段数} 段`  |
| Body    | `（转录前 60 字）...` |
| Actions | `查看` / `复制全文`   |

**触发**：录音的 `transcribeStatus` 由 `running` 转为 `done`，且用户不在主窗口前台（前台时不通知）。

---

#### N2 — 转录失败

| 字段    | 内容                       |
| ------- | -------------------------- |
| 标题    | `{标题} 转录失败`          |
| 副标题  | `{简短错误原因}`           |
| Body    | `点击查看详情和重试选项。` |
| Actions | `查看` / `重试`            |

---

#### N3 — 模型下载完成（首次 / 升级）

| 字段    | 内容                             |
| ------- | -------------------------------- |
| 标题    | `本地转录模型已就绪`             |
| 副标题  | `{model_display_name}`           |
| Body    | `现在可以开始录音并本地转录了。` |
| Actions | `开始录音`                       |

---

#### N4 — 长录音提醒（>2 小时，v0.2 起）

| 字段    | 内容                                           |
| ------- | ---------------------------------------------- |
| 标题    | `已录制 2 小时`                                |
| 副标题  | `（{标题}）`                                   |
| Body    | `LazyAudio 会继续录音，记得在合适的时候停止。` |
| Actions | `停止录音`                                     |

v0.1 不做，仅 spec 占位。

---

#### N5 — App 后台恢复未完成转录

| 字段    | 内容                               |
| ------- | ---------------------------------- |
| 标题    | `恢复 {N} 个未完成的转录`          |
| Body    | `上次未完成的转录已重新加入队列。` |
| Actions | `查看`                             |

仅 App 启动时检测到崩溃恢复时触发一次。

### 2.3 通知开关

设置 → 通用 → 通知（v0.2）：

- [ ] 转录完成
- [ ] 转录失败
- [ ] 模型下载完成
- [ ] 长录音提醒
- [ ] App 后台恢复

v0.1 全部开启，不可关。

## 3. 右键 / 上下文菜单

### 3.1 通用规范

- 尺寸：宽度 240（自适应内容），高度自适应
- 圆角 8，shadow-md
- 项高度 30，padding [0, 12]
- icon 14px 左对齐 + label 12 + 快捷键 11 gray 右对齐
- separator: 1px gray-100
- danger 项：文字 danger 色
- 触发：右键点击 / 长按（macOS trackpad two-finger）

### 3.2 列表项右键菜单

```
┌────────────────────────────┐
│  ▷ 播放                ⏎  │
```

## 4. 内嵌错误状态（Inline Errors）

不弹 dialog，不发系统通知，仅在 UI 局部显示。统称 toast / banner / inline。

### 4.1 顶部 Toast / Banner

详见 [`./main-window-states.md#顶部-toast--banner`](./main-window-states.md)。

### 4.2 表单内联错误

例如 API 配置 input 错误：

- input 边框变 danger
- 下方 12px danger 字：具体错误信息
- icon `alert-circle` 12px

### 4.3 卡片内错误

详情区转录卡 / 摘要卡的失败态，详见 [`./main-window-states.md#状态-5--转录失败`](./main-window-states.md)。

### 4.4 模型下载行内错误

设置 → 转录引擎 → 模型列表中，下载失败的行：

- 进度条变红
- 行尾按钮 `[重试]` 替代 `[下载]`
- 行下方一行 11 danger 字：`下载失败 — {reason}` + `[复制]` 链接
