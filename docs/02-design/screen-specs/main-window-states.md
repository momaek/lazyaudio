## 状态总览

主窗口在 v0.1 有 4 种核心状态变体（不是 4 个独立屏幕，而是同一布局的不同填充）：

| 状态 | 触发 | 已完成 |
|---|---|---|
| 1. 有数据（默认）| 至少 1 条录音存在 | ✅ |
| 2. 空状态 | 无任何录音 | ✅ |
| 3. **录音中** | 有录音任务运行 | **待画** |
| 4. **转录中** | 有转录任务运行 | **待画** |
| 5. **转录失败** | 选中项 transcribeStatus=failed | **待画** |

另有不影响整体布局的"局部状态"（同一屏内的局部变化），用文字 spec 不必单独画大屏：
- 行内重命名编辑态
- 文本选中态
- 摘要生成中
- 摘要失败
- 列表项 hover
- 搜索激活态

## 状态 3 — 录音中（Main — Recording）

### 触发
用户从菜单栏 / 浮窗 / 主窗口 触发录音，且当前正在录。

### 整体变化
和"有数据"态相比的**3 处差异**：

1. **顶部新增 录音中横条**（高 40，bg `#E5484D0F`，bottom-border `#E5484D33`）
2. **列表顶部插入"录音中"列表项**（替换原顶部的"今天"分组头位置）
3. **详情区切换为"录音中"专属面板**（不显示历史录音）

### 1. 顶部录音中横条

宽度 = 主窗口宽 - 标题栏；高 40；padding [0, 20]；horizontal layout space_between：

**左侧 group**（gap 10, alignItems center）：
- 呼吸红点 9×9
- 时长 `03:24` 14px bold monospace record 色
- 分隔 `·` gray-300
- 类型徽章（紫色面试官示例）
- 状态文字 `录制中` 12px record 色

**右侧 button group**（gap 8）：
- `[暂停]` button 32 height，icon `pause` + 文字
- `[停止]` primary record-bg + `square` icon + 文字 `停止并保存`

### 2. 列表中的"录音中"列表项

替代"今天"分组头出现在列表最顶部：

```
┌─────────────────────────────────────┐
│ ● 03:24  面试官 (录制中)             │
│           mic + system  ·  78 MB     │
└─────────────────────────────────────┘
```

- 高度 56（紧凑模式）
- 背景 `#E5484D0A`（极浅红）
- 左侧 3px record 色边
- 上行：呼吸红点（8px，替代类型色点）+ 时长 monospace + 标题 `{类型} (录制中)`
- 下行：mic + system icon + 当前文件大小（每秒更新）

**不可被选中** — 点击只能展开录音中面板，不能切换到其他录音。

### 3. 详情区切换 — 录音中面板

布局 vertical，padding [40, 48]，gap 32，justifyContent center：

#### 顶部 — 类型 + 标题（可编辑）
- 类型徽章（大版）+ 标题 input `面试官 2026-05-16 14:30`（用户可改默认标题）

#### 中部 — 实时电平表（两条）
两个 horizontal group，gap 16：

```
🎤 麦克风     ▓▓▓▓▓▓░░░░░░░░░░  -18 dB
🔊 系统音     ▓▓▓░░░░░░░░░░░░░  -32 dB
```

- 左侧 icon + label 13
- 中间电平条 320×8，bg gray-200，fill accent；fill 实时更新；超过 -3 dB 时变红（峰值警告）
- 右侧 dB 值 11 monospace

#### 中下 — 简易波形预览
一条横向波形条 720×64，bg gray-100，展示最近 30 秒的波形预览（滚动）。不是精细波形，仅给用户"还在录"的视觉反馈。

#### 底部 — 大停止按钮
居中：

- `[暂停 ⌘⇧P]` outline button 40 height
- `[● 停止录音 ⌘⇧R]` record-bg primary button 48 height（更大，强 CTA）

### Mockup 必画
- 录音中（默认）

## 状态 4 — 转录中（Main — Transcribing）

### 触发
列表中至少一项 `transcribeStatus = running`，用户选中该项。

### 整体变化
- 列表项视觉变化：徽章替换为"转录中"徽章
- 详情区不可全部交互，转录文本区为骨架屏，摘要区禁用

### 列表项 — 转录中态

替换原默认列表项布局：

```
┌─────────────────────────────────────┐
│ ⟳ 34%  面试官 2026-05-16 14:30      │
│         01:12:34   预计还需 4 分钟    │
└─────────────────────────────────────┘
```

- 左侧不显示类型色点；显示转录中徽章（旋转的圆环 icon `loader-2` + 进度 `34%`），pending 色
- 标题正常
- 副信息：时长 + ETA `预计还需 X 分钟`
- 选中态：和默认一样的 accent 左边

### 详情区 — 转录中

#### 详情头
- 类型徽章 + 标题正常
- 右上：替代 `[↻ 重新生成]` 显示 `[取消转录]` red outline button
- 状态行：`转录中 · SenseVoice · 已处理 24 / 72 分钟`

#### 播放器
- 完全可用（音频文件已存在）
- 波形完整渲染
- 用户可以一边听一边等转录

#### 转录区 — 骨架屏

不显示真实文字，而是显示骨架占位：

```
[--:--]  ●─  ████████████████  ████████  ██████
[--:--]  ●─  ██████  ████████████████████  ████  ██████
[--:--]  ●─  ████████  ██████████  ███  ████████████
[--:--]  ●─  ██████████████  ████████  ████████
```

- 每行：时间戳占位（gray-200 8 个字符宽）+ speaker 占位（圆 6px gray-200）+ 多个长短随机的横条（高 12，bg gray-200，cornerRadius 4，间距 6）
- 5-7 行铺满，最后一行 fade out 到 0% 透明度
- 顶部一行状态条：`正在转录 · 34% · 24 / 72 分钟`，下面是 8px 高的进度条 accent fill

#### 摘要面板 — disabled
- 整面板 opacity 0.5
- 文字 `等待转录完成后自动生成`
- 空底 — 不显示骨架

#### 顶部 toast（可选）
`转录已开始，可以关闭主窗口，完成时会通知`，gray bg，5s 后自动消失。

### Mockup 必画
- 转录中（默认）

## 状态 5 — 转录失败（Main — Failed）

### 触发
选中项 `transcribeStatus = failed`。

### 列表项 — 失败态

```
┌─────────────────────────────────────┐
│ ⚠ 失败  面试官 2026-05-16 14:30      │
│          01:12:34  转录失败，点击重试 │
└─────────────────────────────────────┘
```

- 左侧 lucide `alert-circle` 14px danger 色
- 标题正常字色 gray-900
- 副信息：时长 + `转录失败，点击重试` danger 色
- 列表项整体 bg `#DC26260A`（极浅红）

### 详情区 — 失败态

#### 详情头
- 标题正常
- 状态行：danger 色 `转录失败 · 2026-05-16 14:30` + `[复制错误]` 链接

#### 播放器
- 完全可用（音频未损坏）

#### 转录区 — 错误卡

居中显示，padding [80, 120]：

```
┌─ 转录失败 ──────────────────────────┐
│  ⚠  网络连接超时                     │
│                                      │
│  转录请求在 60 秒内未收到响应。       │
│  可能的原因：                         │
│  • 网络不稳定                         │
│  • API 服务暂时不可用                 │
│  • 防火墙 / 代理拦截                  │
│                                      │
│  ┌─ 技术详情 ──────────────────┐    │
│  │ Error: ETIMEDOUT          │    │
│  │   at endpoint: ...        │    │
│  │   model: gpt-4o-transcribe│    │
│  │   timestamp: 14:32:01     │    │
│  └────────────────────────────┘    │
│  [折叠] [复制错误]                   │
│                                      │
│  [重试]   [换引擎]   [跳过 — 不转录]  │
└──────────────────────────────────────┘
```

- danger icon 24px + 简短原因（一句话） 16px medium
- 详细说明 13px gray-700，可能原因 bullet 列表
- "技术详情"默认折叠，展开后显示堆栈 / endpoint / model / 时间戳，可复制
- 底部三个按钮：
  - `[重试]` primary（用当前引擎）
  - `[换引擎]` secondary（切到另一引擎再试，如本地 → 云端）
  - `[跳过 — 不转录]` text link gray，确认后将 transcribeStatus 永久标 `skipped`

#### 摘要面板
disabled，文字 `转录失败，无法生成摘要`。

### 错误类型映射（影响错误卡内容）

| 错误类型 | 简短原因 | 可能原因列表 |
|---|---|---|
| 网络超时 | `网络连接超时` | 网络不稳 / API 不可用 / 防火墙 |
| 401 | `API key 无效或已过期` | API key 错 / 账号失效 |
| 429 | `请求过于频繁` | 触发 rate limit / 余额耗尽 |
| 500 | `API 服务异常` | 服务端问题，建议稍后重试 |
| 模型未下载 | `本地模型未就绪` | 模型被删 / 路径错 |
| 模型崩溃 | `转录引擎崩溃` | 内存不足 / 文件损坏 |
| 文件损坏 | `音频文件损坏或不完整` | 录音中崩溃 / 磁盘问题 |
| Worker timeout | `转录超时` | 录音过长 / 设备性能不足 |

### Mockup 必画
- 转录失败（默认）

## 局部状态（不需要单独大屏，仅文字 spec）

### 列表项 hover

- 背景从 transparent → `surface/hover`（gray-100）
- 右侧出现 `⋯` icon button 14px，点击 = 弹右键菜单

### 列表项 — 行内重命名

- 双击标题 / 按 Enter 触发
- 标题文字变为 text input：30 高，padding [0, 8]，accent ring focus
- Enter / blur 保存；Esc 取消
- 保存后 input 还原为文字，触发 meta.json 更新

### 转录文本 — 选中态

- 选中字符背景 `accent @ 20%`，文字色不变
- 跨多段选择时跨段连续高亮
- `⌘C` 复制纯文本（不带时间戳）；`⌘⇧C` 复制带时间戳

### 转录文本 — 当前播放段

- 当前播放的段：背景 `accent @ 8%`，时间戳 + speaker 颜色变 accent + 加粗
- 自动滚动到视口中央偏上 1/3 处
- 用户手动滚动后 5s 内不自动跟随，5s 后恢复

### 摘要面板 — 生成中

- 摘要 body 显示骨架（同转录骨架风格）
- 顶部状态：`正在生成（候选人评估模板）…`
- 底部 actions disabled

### 摘要面板 — 生成失败

- 摘要 body 显示小错误卡（简化版的状态 5 错误卡）
- 顶部 actions `[重试]` `[换模板]`

### 搜索激活态

- 顶部搜索框 focus：accent ring
- 列表自动过滤，命中文字 yellow @ 30% 高亮
- 列表分组头根据过滤结果隐藏（无结果分组不显示）
- 无任何结果：列表区中央显示 `未找到匹配的录音` + `[清除搜索]` 链接

### 顶部 toast / banner

非阻塞通知，3-5 秒自动消失：

| 触发 | 文案 | 颜色 |
|---|---|---|
| 音频设备切换 | `音频设备切换为 {device_name}` | gray |
| 磁盘空间预警（< 1 GB） | `磁盘空间不足 1 GB，可能影响录音` | warning |
| 自动停止（长时间静音） | `检测到 60 秒静音，已自动停止录音` | gray |
| 网络恢复（云端模式 retry） | `网络已恢复，转录任务恢复` | success |

视觉：高 36，宽 fit-content，居中或顶部偏右，圆角 8，shadow-md，icon + 文字 + × 关闭。

## 跨状态一致性

| 项 | 规则 |
|---|---|
| 状态切换 | 100ms ease-out fade，不做大幅 motion |
| 错误状态 | 永远提供 "重试" + "替代方案" 两个出路 |
| 加载状态 | 永远有进度感（百分比或 ETA），避免单纯 spinner |
| 时间戳 | monospace 11px gray-500（hover：accent 加粗）|
| 录音相关红色 | 仅录制态使用，其他状态用 danger 而非 record |
| 详情区切换 | 选中列表项 → 详情区无缝替换，不刷新整个右栏 |
