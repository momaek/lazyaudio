# LazyAudio — 产品需求草稿 v0.1

> 这是一份讨论用的活文档。`TBD` 标记的是还没拍板的点。

## 一句话定位

一个本地优先的桌面应用，**录下你电脑的音频和麦克风音频，转录成可搜索的文字，并能用 LLM 提炼要点**。

## 目标用户和场景

| 优先级 | 场景 | v0.1 关注度 |
|--------|------|-------------|
| P0 | 线上会议（Zoom / Teams / 腾讯会议）| 主要打磨对象 |
| P0 | 语音笔记 / 灵感速记 | 快捷键 + 速度 |
| P0 | 面试 — 我作为**面试官** | 复盘候选人回答、生成结构化评估 |
| P0 | 面试 — 我作为**面试者** | 复盘自己答得怎样、记下没答好的题 |
| P1 | 课程 / 讲座录制 | 能跑就行，长录音稳定性 |
| P2 | 播客 / 内容创作 | 不优化质量上限，但分轨保留可能性 |

### 场景间的共性与差异

| | 会议 | 笔记 | 面试官 | 面试者 | 课程 | 播客 |
|---|---|---|---|---|---|---|
| 默认音源 | mic + 系统音 | 仅 mic | mic + 系统音 | mic + 系统音 | 系统音为主 | mic 为主 |
| 典型时长 | 30min–2h | <5min | 30min–1h | 30min–1h | 1–3h | 30min–2h |
| LLM 模板 | 会议纪要 + 待办 | 要点速记 | 候选人评估 | 自我复盘 | 章节笔记 | 章节标记 |
| 多录音的对比维度 | — | — | **横向**（候选人对比） | **纵向**（自己历次） | — | — |
| 协作 | 可分享 | 私人 | 团队分享 | **绝对私人** | 私人 | 公开/分享 |

## 平台

- v0.1：macOS（优先，自己狗粮）+ Windows
- v0.x：Linux 视需求

## 技术栈（已定）

- Electron + Vite + React + TypeScript
- pnpm
- 本地转录：sherpa-onnx（中文场景考虑 SenseVoice）
- 云端转录：OpenAI 兼容 API（让用户填 baseUrl / apiKey / model）
- 系统音采集：
  - macOS 13+：ScreenCaptureKit（需要写 Native Node Addon 或用现成的 npm 包）
  - macOS <13：引导用户装 BlackHole 类虚拟声卡
  - Windows：WASAPI loopback（Electron 的 `desktopCapturer` 可直接拿，或写 native）

## 核心交互形态

- **独立主窗口 + 菜单栏图标**
- 菜单栏：开始/停止录音、最近一次录音状态、快速进入主窗口
- 主窗口：左侧录音库（按时间倒序），右侧详情（音频播放器 + 转录文本 + LLM 摘要面板）
- **全局快捷键**：开始/停止录音（默认 TBD，比如 `⌘⇧R` / `Ctrl+Shift+R`），可在设置中改

## 首次启动 Onboarding

1. 欢迎页
2. **隐私模式选择**：本地（推荐，需下载模型）/ 云端（需填 API key）
3. 权限授权引导：
   - macOS：麦克风权限 + 屏幕录制权限（用于 ScreenCaptureKit 抓系统音）
   - Windows：麦克风权限
4. 快捷键确认 / 录音保存目录确认
5. **录音合规告知**：一次性提示"在部分国家/地区录制对方音频前需告知对方，请遵守当地法律"。设置中可关闭此提示
6. 完成 → 进入主窗口

## v0.1 功能范围（MVP）

### 必须有

- [x] **录前选择会话类型**：**通用**（默认）/ 会议 / 笔记 / 面试-面试官 / 面试-面试者 / 课程 / 播客。每种类型预设默认音源和 LLM 模板，可改、可记住"上次选择"；"通用"音源默认 mic+system 都开，LLM 模板用通用要点提炼
- [x] 同时录制 **系统音 + 麦克风**，可在录前选择（默认音源由会话类型决定）
- [x] 录音过程中显示：录制时长、电平表、可暂停 / 停止
- [x] 录音文件保存到本地（**分轨：mic.wav + system.wav，同时生成 mixed.wav**——TBD 是否同时生成混音）
- [x] 录音结束自动触发转录（可在设置里关掉）
- [x] 录音库列表：**按日期分组**（今天 / 昨天 / 本周 / 本月 / 更早），组内按时间倒序；顶部有搜索框 + 类型筛选 chip；每条显示「类型徽章 + 标题 + 时长 + 转录预览首句」；支持重命名、删除、在 Finder/Explorer 中显示
- [x] 录音的**默认标题**：`{会话类型} {YYYY-MM-DD HH:mm}`（如 `面试官 2026-05-16 14:30`），用户可随时改
- [x] 转录文本展示，**点文字跳到对应音频位置**
- [x] 转录文本全文搜索（跨所有录音）
- [x] 复制单段 / 整段文本
- [x] 全局快捷键开始/停止
- [x] 菜单栏图标 + 录制状态指示

### 应该有（v0.1 力所能及就做）

- [ ] LLM 摘要面板：选定一段录音 → 调用配置的 OpenAI 兼容 API → 生成总结
  - 预置模板（按会话类型自动套用，也可手动切换）：
    - **会议纪要**：议题、决议、待办、悬而未决的问题
    - **要点速记**：散乱笔记整理成结构化要点
    - **候选人评估**（面试官）：分维度评分（技术/沟通/思考过程）、亮点、风险点、推进建议
    - **自我复盘**（面试者）：被问到的问题清单、自己的回答要点、卡壳点、改进建议
    - **章节笔记**（课程）：按主题分段、关键概念、可能的考点
  - 用户可自定义模板（v0.1 至少支持改 prompt）
  - **TBD**：v0.1 是否包含？建议算
- [ ] 导出转录文本（md / txt / srt）

### 不做（推迟到 v0.2+）

- 实时流式转录（架构预留接口）
- 说话人分离（diarization）——但分轨录制天然有 mic/system 两个"伪说话人"标签
- 多麦克风源混录
- 云端同步
- 转录文本编辑（修正错字）
- 标签 / 文件夹层级
- **候选人横向对比**（面试官场景）——非核心，外围工具或后续版本再考虑

## 数据模型（草稿）

```
~/Library/Application Support/LazyAudio/   (macOS)
├── settings.json
├── recordings/
│   └── 2026-05-16T14-30-00_meeting/
│       ├── meta.json          # 见下方字段
│       ├── mic.wav
│       ├── system.wav
│       ├── mixed.wav          # 可选
│       ├── transcript.json    # 段落 + 时间戳 + speaker(mic/system)
│       └── summary.md         # LLM 总结结果（按选用的模板生成）
├── templates/                 # 用户自定义的 LLM 模板
└── models/                    # sherpa-onnx 模型缓存
```

meta.json 字段：
```ts
{
  id: string
  title: string                // 默认 "{sessionType} {date}"，用户可改
  sessionType: 'general' | 'meeting' | 'note' | 'interview-as-interviewer'
              | 'interview-as-candidate' | 'lecture' | 'podcast'
              // 'general' 是默认值，用户不想分类时就用它
  startedAt: number            // unix ms
  durationMs: number
  sources: { mic: boolean; system: boolean }
  llmTemplate?: string         // 用了哪个模板生成 summary
  tags?: string[]              // v0.2 可能用
}
```

## 已拍板的决定（按确认顺序）

- 平台：macOS（优先）+ Windows
- 技术栈：Electron + Vite + React + TypeScript + pnpm
- 本地转录：sherpa-onnx；云端：OpenAI 兼容 API
- 形态：独立窗口 + 菜单栏图标
- 转录时机：v0.1 离线，架构预留实时接口
- 全局快捷键：要
- 录音文件格式：用户可选（默认 TBD，见下）
- 隐私模式：首次进入让用户选
- 面试场景（面试官 + 面试者）：P0，和会议/笔记同等
- 会话类型元数据：录前选择，预置 6 种
- 录音合规提醒：首次启动一次性提示 + 设置可关
- 录音库组织：按日期分组，标题前缀显示类型，顶部支持类型筛选 + 全文搜索
- 默认本地转录模型：**SenseVoice**（中文友好、速度快、约 220MB）
- 候选人横向对比功能：**不做**，外围工具或后续版本再考虑，不属于核心
- 录音文件：**默认同时生成分轨（mic.wav + system.wav）和混音（mixed.wav）**，分轨可在设置里关
- 全局快捷键：用户可在设置中自定义；**默认值待 02-design 阶段定**（候选 `⌘⇧R` / `Ctrl+Shift+R`）
- 菜单栏录制状态：**时长 + 呼吸红点**（如 `● 03:24`，红点带轻微透明度脉冲）
- App 名：**LazyAudio**（v0.1 暂用，Icon 占位即可）
- 会话类型：**通用（默认）+ 会议 / 笔记 / 面试-面试官 / 面试-面试者 / 课程 / 播客**，共 7 种；"通用"是兜底选项，避免用户卡在"选哪个"
